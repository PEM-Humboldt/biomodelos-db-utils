Geoserver

Configuración de GeoServer para Biomodelos y carga de modelos.

Examples

Configurar conexión a GeoServer

bmdbutils geoserver setup --url http://localhost:8080/geoserver --username admin --password admin

Cargar un nuevo modelo a Geoserver.

bmdbutils geoserver upsert model_file.csv .

Parámetros

Usage: bmdbutils geoserver [OPTIONS] COMMAND [ARGS]...

Options:
  --help  Show this message and exit.

Commands:
  setup   Configurar GeoServer
  upsert  Crear o actualizar los archivos de modelos en el GeoServer para
          modelos existentes en BioModelos

Consideraciones

  • setup
    • No es necesario si ya se ha realizado la configuración previamente. Sus valores por defecto apuntan al servidor de producción.
    • La URL del localhost debe ser escrita con el prefijo http.
  • upsert
    • Se debe contar con un CSV con el id de la especie (tax_id), el id del modelo de Geoserver (model_id) y el nombre del archivo .tif a cargar (model_file)
    • El archivo TIF debe estar en la carpeta especificada en el segundo argumento.
    • No es necesario agregar el parámetro --password, éste se solicita automáticamente

Comportamiento del comando

  • upsert
    • El comando carga el modelo al GeoServer previamente configurado mediante setup. En caso de no configurarlo, se asume el GeoServer de producción por defecto.
    • El comando valida la existencia del archivo(s) TIF especificado en el archivo CSV.
    • El comando busca el archivo(s) TIF especificado en el directorio especificado en el segundo argumento del comando.
    • El comando carga el modelo al Geoserver previamente especificado con el comando setup.
    • El comando evita duplicados. Si el modelo ya existe, retorna error.

Valores por defecto

  • setup
    • --api-url: URL de producción http://192.168.11.58:8181/geoserver
    • --username: Usuario de GeoServer de producción (admin)

⚠️ Estos valores corresponden al entorno de producción y deben usarse con precaución.
